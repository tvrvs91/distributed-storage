Это децентрализованная распределённая система для хранения файлов, написанная на Go. Система состоит из нескольких узлов, которые автоматически синхронизируют данные между собой и обеспечивают отказоустойчивость.
Архитектура
Система построена по принципу peer-to-peer (P2P), где каждый узел является равноправным участником сети. Когда файл загружается на любой узел, он автоматически реплицируется на другие узлы для обеспечения надёжности. Если один или несколько узлов выходят из строя, система продолжает работать, так как файлы хранятся на нескольких узлах одновременно.
Требования
Для запуска системы тебе понадобится установить Docker и Docker Compose. Также убедитесь, что у вас установлен Go версии 1.21 или выше (если хочешь запускать систему без Docker).

1. docker-compose up --build (для сборки и запуска всех узлов:  8081, 8082, 8083 и 8084 порты)
2. curl http://localhost:8081/health
   curl http://localhost:8082/health
   curl http://localhost:8083/health
   curl http://localhost:8084/health
   Каждый узел должен ответить сообщением {"status":"healthy"}
3. При загрузке файла на один из узлов, система автоматически реплицирует его на остальные узлы

   echo "Привет, это тестовый файл!" > test.txt            |   загрузка файла на 8081
   curl -F "file=@test.txt" http://localhost:8081/upload   |

   curl http://localhost:8081/list                         |   проверка на содержание в других узлах
   curl http://localhost:8082/list                         |
4. Загруженный на один из узлов файл можно скачать с любого узла

   curl http://localhost:8082/download/test.txt -o downloaded.txt
   cat downloaded.txt
